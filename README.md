# La Costa Hotel — Investor Website

Single-page marketing/investor site for **La Costa Hotel** (The Brown Studio),
a 17-key boutique hotel + destination restaurant + bungalow/row-house
neighborhood at 516 La Costa Ave, Leucadia/Encinitas, CA.

Built from two source decks:
1. **Lender deck** — "Lender Version - La Costa Hotel - Brown Studio" (60-slide
   Google Slides deck: market data, positioning, sales & marketing, experience,
   full pro forma / financials).
2. **Brand lookbook** — `LA_COSTA_-_BRAND_BOOK_20260529-11x17.pptx` (15-slide
   PowerPoint: real photography, renderings, floor plans, brand identity).

The site intentionally includes **every section of the 60-slide deck** —
market data, STR charts, whitespace matrix, guest profiles, inspiration set,
sales/marketing strategy, full pro forma tables, restaurant P&L, etc. Nothing
was cut for length. If asked to "add anything missing," check against the
deck's Part 1–6 structure below before assuming something is new.

## Files

```
index.html              ← EDIT THIS. Source file, images as separate assets/*.jpg
build_standalone.py     ← Run after editing to regenerate the shippable file
index_standalone.html   ← GENERATED. Self-contained, base64-inlined, ~1.8MB.
                          This is the file that actually gets delivered/shared.
assets/                 ← 22 images (real photography + renderings + floor plans,
                          pulled from the brand lookbook and lender deck)
```

**Workflow: edit `index.html` → run `python3 build_standalone.py` → deliver
`index_standalone.html`.** Never hand-edit the standalone file; regenerate it.

## Design system

Single `<style>` block in `index.html`, CSS custom properties at the top:

```css
--cream: #F7F1E4        /* main background */
--cream-2: #FBF7EE       /* cards, nav-solid bg */
--umber: #2B1A13         /* primary text, dark section bg */
--clay: #83663A          /* accent (eyebrows, chart lines, links) */
--sage-deep: #545E42     /* alternating section bg */
--navy: #1E3C75          /* unused currently, reserved */
```

Fonts (Google Fonts, loaded via `<link>` in `<head>`):
- **Fraunces** — all headings/display type
- **Josefin Sans** — nav, eyebrows, small caps labels, chart axis labels
- **Lato** — body copy

Signature motif: a hand-drawn "sprig" recreated as inline SVG (`.sprig` class,
used in nav logo + footer) — matches the botanical line-art icon from the
brand lookbook's logo lockup.

Section rhythm alternates `cream` (default) → `dark` (`.pad.dark`, umber bg,
used for Market) → `sage` (`.pad.sage`, used for Positioning / Strategy /
closing quote) → back to cream. Keep this alternation if adding sections.

## Naming note

All references to the restaurant concept were originally "Manna/Amagi" or
"Atelier Manna" (Chef Andrew Bachelier's concept name from the deck). **Per
Ivan's instruction, every instance was replaced with "The Restaurant" /
"Restaurant"** (dining eyebrow, chef bio copy, pro forma table caption,
"Why La Costa Wins" copy, footer link). If new restaurant-related content is
added, keep using "The Restaurant," not the original concept name, unless
told otherwise.

## Interactive charts (Market section)

STR occupancy/ADR/RevPAR charts are hand-rolled inline SVG generated by JS
(`renderChart()` near the bottom of `index.html`), not a charting library —
this was a deliberate choice for portability (no CDN dependency, works
offline, renders identically everywhere). Monthly data (Dec 2024–May 2026)
for both comp sets lives in the `STR` object in the `<script>` block. Tab
buttons (`.chart-tabs`) swap between occupancy/ADR/RevPAR by re-rendering the
SVG — extend `STR` and `MONTHS` if the data range grows.

## Known gotchas (learned the hard way — don't rediscover these)

- **Full-page screenshot tools break on this page's height.** The page is
  ~35,000px tall. `wkhtmltoimage` doesn't support IntersectionObserver or CSS
  Grid at all (renders `.reveal` content invisible and stacks grid columns) —
  don't use it for QA. Playwright Chromium's `full_page=True` screenshot also
  silently blanks out large chunks on pages this tall (a real Chromium
  texture-size limit, not a site bug — confirmed via
  `getBoundingClientRect()` that the DOM layout is correct). **For QA, scroll
  and screenshot the viewport in ~850px increments instead of one full-page
  capture.**
- **Reveal-on-scroll animation** (`.reveal` / `.reveal-stagger`, driven by
  IntersectionObserver) defaults every element to `opacity:0` until JS adds
  `.in`. When QA'ing with a tool that doesn't run JS/observers properly,
  temporarily patch the CSS (`opacity:0` → `opacity:1`, `transform:...` →
  `transform:none`) in a throwaway copy — don't edit the real file for this.
- **Tables can be wider than their grid column** (e.g. the 4-column Financing
  table). This is intentional — `.table-scroll{overflow-x:auto}` lets them
  scroll horizontally rather than break the layout. Not a bug if a table
  looks "cut off" in a screenshot at a fixed viewport width.
- **pip installs** need `--break-system-packages` in this environment.
  **Playwright** (`import playwright.sync_api`) is available and is the
  right tool for visual QA; install Chromium with
  `playwright install chromium` if it's not already present.

## Content sourcing (if you need to pull more from the original decks)

- Lender deck was a Google Slides doc; the PDF export in the project files is
  actually **a zip archive of per-slide JPEG+TXT pairs**, not a real PDF —
  use `unzip`, not PDF tooling, if re-extracting.
- Brand lookbook (`LA_COSTA_-_BRAND_BOOK_20260529-11x17.pptx`) was unpacked
  with the pptx skill's `unpack.py`, then rendered slide-by-slide via
  LibreOffice → `pdftoppm` to visually identify real photography vs.
  decorative assets before pulling specific `ppt/media/imageN.jpg` files.
  Most of `assets/` (hero shot, site plan renders, interior photography,
  floor plans) came from this process, not the lender deck's low-res
  embedded thumbnails.
